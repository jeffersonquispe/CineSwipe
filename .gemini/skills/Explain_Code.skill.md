# Skill: Explain Code

## Objetivo
Traducir lógica técnica compleja a lenguaje accesible para desarrolladores de cualquier nivel, manteniendo el rigor técnico pero priorizando la comprensión del "por qué" sobre el "qué".

## Estructura Requerida de Respuesta
Cada vez que se invoque este skill, la explicación debe seguir este orden:

1. **Contexto de Alto Nivel**: Explicar qué problema resuelve el código y dónde encaja en la arquitectura (máximo 3 líneas).
2. **Desglose Modular**: Si el código es extenso, dividirlo en bloques lógicos.
3. **Diagrama de Flujo (Mermaid)**: Visualizar la interacción de datos o componentes.
4. **Guía de Uso (Snippet)**: Mostrar el código "en acción" con el escenario más común.
5. **Anti-patrones y Consejos (Tabla)**: Listar 3 errores comunes que un junior podría cometer al usar este código y cómo evitarlos.

## Principios de Comunicación
- **Pedagogía sobre Presunción**: No asumir que el desarrollador conoce las APIs internas.
- **Visualización**: Usar diagramas Mermaid siempre que haya más de 2 hilos de lógica interactuando.
- **Alertas de Importancia**: Usar bloques `> [!NOTE]` para datos curiosos y `> [!WARNING]` para riesgos de seguridad o performance.
